# Amiga DB dump for staging import

Generated by `scripts/export_ko2amiga_db.ps1` before WinSCP sync. SQL part files are **gitignored**; WinSCP still carries them with `public_html/`.

| File | Role |
|------|------|
| `ko2amiga_manifest.json` | Part list (16 files); importer reads this |
| `ko2amiga_01_schema.sql` | DDL (drop + create A2 tables) |
| `ko2amiga_02_tournaments.sql` … `ko2amiga_03_players.sql` | Ground-truth data |
| `ko2amiga_04_games_*` … `ko2amiga_15_ratings_*` | Games + ratings in ~5k-row chunks |
| `ko2amiga_16_stats.sql` | Derived player stats |
| `ko2amiga_db.sql` | Optional full dump (Heidi / mysql CLI fallback) |

**Import on staging (verified Jun 2026):** `../run_import_ko2amiga.php`

- **Preview:** `/amiga/run_import_ko2amiga.php?once=ko2amiga-import-one-shot&pwd=coffee` — expect `parts: 16`
- **Apply:** same URL with `&apply=1&part=1` (auto-continues through part 16)

Password **`coffee`** in URL or on the form if `pwd` is omitted. Full loop: [`docs/amiga-staging-handoff.md`](../../../../docs/amiga-staging-handoff.md).

This folder is not web-accessible (`.htaccess`); the import PHP reads files from disk.
