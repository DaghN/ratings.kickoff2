# Amiga DB dump for staging import

Generated by `scripts/export_ko2amiga_db.ps1` before WinSCP sync. SQL part files are **gitignored**; WinSCP still carries them with `public_html/`.

| File | Role |
|------|------|
| `ko2amiga_manifest.json` | Part list; importer reads this |
| `ko2amiga_01_schema.sql` | DDL (drop + create Amiga tables) |
| `ko2amiga_02_format_templates.sql` … `ko2amiga_08_fixtures.sql` | Format/template + ground-truth data |
| `ko2amiga_09_games_*` … `ko2amiga_*_ratings_*` | Games + per-game ratings in ~5k-row chunks |
| `ko2amiga_*_event_snapshots.sql` | Player×event snapshot rows (career + event-local) |
| `ko2amiga_*_player_current.sql` | Present career projection per player |
| `ko2amiga_*_standings.sql` … `ko2amiga_*_catalog_stats.sql` | Derived tournament standings/index stats |
| `ko2amiga_*_matchup_summary.sql` … `ko2amiga_*_finish_override.sql` | H2H summary, generalstats, Tier E overrides |
| `ko2amiga_db.sql` | Optional full dump (Heidi / mysql CLI fallback) |

**Import on staging (verified Jun 2026):** `../run_import_ko2amiga.php`

- **Preview:** `/amiga/run_import_ko2amiga.php?once=ko2amiga-import-one-shot&pwd=coffee` — expect manifest part count from latest export
- **Apply:** same URL with `&apply=1&part=1` (auto-continues through all parts)

Password **`coffee`** in URL or on the form if `pwd` is omitted. Full loop: [`docs/amiga-staging-handoff.md`](../../../../docs/amiga-staging-handoff.md).

This folder is not web-accessible (`.htaccess`); the import PHP reads files from disk.
