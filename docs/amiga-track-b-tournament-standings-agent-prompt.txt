INVESTIGATION FINDINGS (locked ‚Äî verify with queries, do not re-litigate without new evidence)

Scale (local ko2amiga_db, Jun 2026):
- 604 tournaments, 27,408 games, 473 players
- 61% of games have phase = NULL (informal / open-play events)
- 39% have structured phase labels (cups, group stages, knockouts)

Tournament format signals (tournaments catalog):
| is_cup | equal_teams | ~tournaments | ~games | phase usage |
|--------|-------------|-------------|--------|-------------|
| 0      | 0           | 141         | 6,471  | mostly NULL phase |
| 0      | 1           | 433         | 20,623 | mix NULL + rare phases ‚Äî **marathon round-robins** (e.g. London XXIII: 24 games/player, no phase) |
| 1      | 0           | 5           | 57     | small cups |
| 1      | 1           | 25          | 257    | **structured World Cups / major cups** ‚Äî phase always set |

Phase label taxonomy (non-exhaustive; strings are messy ‚Äî normalize carefully):
- **Group stages:** `Round 1 - Group A` ‚Ä¶ `Round 1 - Group D`, `Round 2 - Group E/F`, `Silver Cup - Group G/H`, `Bronze Cup - Group I`, variants without hyphen (`Round 1 Group A`)
- **Knockout:** `Quarter Finals`, `Semi Finals`, `Final`, `3rd Place Final`, `9th Place Final`, ‚Ä¶
- **Placement brackets:** `Places 9-16`, `Places 17-20`, `Places 25-32`, ‚Ä¶
- **Informal:** phase NULL ‚Äî treat as single-table round-robin within the tournament

Points system (confirmed from Access `Tables`, e.g. London XXIII):
- **3 points per win, 1 per draw, 0 per loss** (W√ó3 + D√ó1 = Pts)
- Goal difference / goals scored as tie-breakers (infer ordering from Access rows when in doubt)

Access reference standings (parity only ‚Äî do NOT import as derived truth):
- `Tables` ‚Äî 4,359 rows, one overall table per tournament (G/W/D/L/GS/GC/Pts/position)
- `World Cup I` ‚Ä¶ `XXIII Tables` ‚Äî per-group snapshots within a cup (multiple `P=1` rows = different groups/stages)
- Normalize into one `amiga_tournament_standings` (or companion tables you register in the contract)

**Critical ground-truth gap ‚Äî `Scores.Extra` NOT imported today:**
- Access column `Extra` on 108 games (~0.4%), mostly World Cup knockouts / placement finals
- Examples: `7-7 e.t.`, `3-5 e.t.`, `(4-4) 5-3 p.k.`, `(5-4 pen.)`, `(0-0) 7-6pen`
- Regulation goals (`goals_a` / `goals_b`) can be a **draw** while `Extra` records ET or penalties
- Elo replay correctly uses regulation score (draw = 0.5 actual_score) ‚Äî **standings/bracket advancement must use match winner**, which may differ from regulation result
- `import_access.py` reads `Extra` in SQL but **drops it** on `AccessScore` ‚Äî must add to ground truth before knockout advancement is reliable

Knockout draws in phase labels:
- ~158 games with phase containing `Final` and regulation draw ‚Äî expect `Extra` or duplicate legs to resolve winner
- Some knockouts appear **twice** (home/away legs or rematches) ‚Äî same players, swapped sides; bracket logic must not double-count as independent ties

Multi-competition events (World Cups):
- Parallel tracks: main World Cup groups, Silver Cup, Bronze Cup (phase prefix distinguishes)
- Staged flow: Round 1 groups ‚Üí Round 2 groups ‚Üí knockouts ‚Üí placement finals
- External human reference (optional parity, not authority): Alkis Polyrakis event pages, e.g. https://www.alkis.org/ecups.html and per-event match pages like https://www.alkis.org/matches11.html ‚Äî useful to understand multi-cup structure, placement finals, and narrative; **not** a machine API; spot-check only

Milan X / alias handling already in tournament_names.py ‚Äî phase may be inferred from fragment tournament names.

DIFFICULTY TIERS (implement in this order)

**Tier 1 ‚Äî Leagues / open round-robins (easy)**
- Tournaments where all games share one logical table: phase NULL, or single implicit group
- Examples: London XXIII, Reading XIII, Gloucester I (200‚Äì300 games each, no phase)
- Output: one standings row per player per tournament (position, G/W/D/L, GF/GA, Pts)
- Parity: compare to Access `Tables` for same tournament name

**Tier 2 ‚Äî Single-group / labeled group stages (medium)**
- Parse `phase` ‚Üí `(competition_track, stage, group_id)` ‚Äî design a normalizer; labels are inconsistent
- Aggregate W/D/L within each group scope
- Output: standings per `(tournament_id, group_key)` e.g. `Round 1 - Group A`, `Silver Cup - Group G`
- Parity: Access `World Cup * Tables` for selected cups

**Tier 3 ‚Äî Knockout brackets + advancement (hard)**
- Requires ground-truth winner per knockout game (regulation winner OR parse `Extra`)
- Placement brackets (`Places 9-16`, etc.) are mini-knockouts ‚Äî same machinery
- May need `amiga_match_decision` ground column or `extra` text on `amiga_games` + parsed `winner_id` for standings only (do not change Elo inputs)
- Handle two-leg ties / duplicate rows ‚Äî investigate per tournament before coding assumptions

**Tier 4 ‚Äî Cross-stage promotion (hardest ‚Äî defer if needed)**
- Rules for who advances from Round 1 groups ‚Üí Round 2 groups ‚Üí WC vs Silver Cup
- May require tournament **format metadata** not fully in DB today (`equal_teams`, `player_count`, `is_cup` are hints only)
- v1 can ship Tiers 1‚Äì2 without simulating full WC draw engine; document gaps

- scope_type: `overall` | `group` | `placement` (agent may refine)
