Implement Amiga chronology fix — calendar-first game order, continuous same-day timestamps, simplify consumers to `ORDER BY game_date ASC, id ASC`.

GOAL
Fix Amiga game ordering so timeline matches tournament calendar dates (not legacy chrono-first), eliminate same-day timestamp resets that break rating charts, and align read paths with online habit: order is materialized at import on each row; replay/API/ops read `game_date, id`.

Phase A1 contract today:
- Import sorts: chrono → event_date → source_scores_id
- Synthetic game_date: tournament event_date midnight + 1s per game WITHIN tournament (resets per tournament → ~79 backward same-day timestamps)
- Replay, PHP post-game, player_rating_history API all re-derive chrono-first order (do not trust id/game_date)
- Access source: 27,408 games, 604 tournaments; no per-game clock time

Known data quirks (from analysis):
- Only 2 adjacent chrono/date inversions in 604 tournaments (World Cup VIII, Newent XVI) — but ~8.8% of games reposition under calendar-first order
- WC VIII: event_date 2008-09-08, chrono 325, processed after many Sep–Nov 2008 events
- Same-day multi-tournament days: 91 calendar days, ~4554 games

Principle (target):
  Import → define order once, write game_date + id
  Replay → ORDER BY game_date, id → derived ratings
  API/UI → ORDER BY game_date, id
