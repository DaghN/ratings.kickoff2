Implement Amiga ops simul (full-history game-by-game) per docs/amiga-data-contract.md — online Mode A analogue.

GOAL
Prove that N sequential calls to ProcessCompletedGame (Amiga), from empty derived state, reproduce the same end state as batch replay:

  zero-derived  ->  simul (game 1..N in contract order)  ->  amiga_game_ratings + amiga_player_stats
  zero-derived  ->  python -m scripts.amiga replay         ->  same derived state

Parity rule: after simul through game G (inclusive), every amiga_game_ratings row and every amiga_player_stats row must match a full Python replay. Spot-check + row-count gates are acceptable for smoke; full diff or sampled hash for sign-off.

Online analogue (pattern only — do not wire into online ops):

  zero-derived     ->  ops/run_prepare.php zero-derived
  replay-to / sim  ->  k2_ops_replay_post_game() in ops/modules/process_completed_game.php
                      ops/run_process_game.php replay-to
                      ops/run_ops_sim.php (Mode C adds FinalizeUtcDay — NOT needed for Amiga v1)

Amiga has no UTC day tick, leagues, milestones, or entered_arena in v1 sim scope.

CURRENT STATE (ProcessCompletedGame v1 done)
- amiga_process_completed_game() writes amiga_game_ratings + amiga_player_stats incrementally
- CLI: php site/public_html/amiga/ops/run_process_game.php process-one --game-id=N
- Idempotent: already_processed if rating row exists
- **Blocker for simul:** v1 enforces append-only — game must be chronologically LAST in amiga_games. That works for one live new game, not for replaying game 2..27408. Simul requires a v2 rule (see CHRONOLOGY below).
- Batch oracle: python -m scripts.amiga replay (~27k games, ~10s local)
- Ground: 27,408 games, contract order = tournaments.chrono, event_date, source_scores_id, g.id

V1 SIMUL SCOPE (this PR)
In scope:
1. **zero-derived** — clear amiga_game_ratings + amiga_player_stats only; never truncate amiga_games / amiga_players / tournaments (mirror replay.py clear_derived)
2. **Sim loop** — e.g. amiga_ops_replay_post_game(mysqli $con, ?limit, ?untilGameId, dryRun) listing games in contract order, one amiga_process_completed_game per id
3. **v2 chronology rule for sim** — replace append-only with “next unprocessed in contract order” when running inside sim (or always: first game without rating row when walking chrono order; all prior rated). Keep append-only for standalone process-one live path OR document that live process-one = sim with one game at end only
4. **CLI** — extend run_process_game.php or add run_amiga_sim.php:
     zero-derived
     replay-to [--limit N] [--until-game-id G] [--dry-run]
     (optional) verify — row counts + sample rating/stats diff vs replay oracle
5. **Progress logging** — [amiga-ops] every 500–5000 games; final summary processed / skipped / skip_reasons
6. **Parity proof** — at minimum:
     zero-derived -> replay-to --limit 500 -> compare to replay --limit 500
     zero-derived -> full replay-to -> row counts + spot-check last game + 2–3 players vs full replay

CHRONOLOGY (critical)
Contract game order (same as replay.py GAME_SELECT):

  ORDER BY COALESCE(t.chrono, 999999) ASC,
           COALESCE(t.event_date, '1970-01-01') ASC,
           g.source_scores_id ASC,
           g.id ASC

Sim must list game ids in this order — NOT amiga_games.id ASC alone.

**v1 live rule (keep):** process-one on a single new game at the end of history.

**v2 sim rule (implement):** game G is processable when:
  - no amiga_game_ratings row for G (else already_processed skip)
  - G is the first game in contract order without a rating row
  - equivalently: count(ratings) + 1 = position of G in ordered list, or derived_gap if a hole exists

Do not use append-only (is last game in table) inside sim loop.
