currently fails globally for local/generated fixture-backed tournaments created before `tournament_entrants` existed. Examples include tournaments `610`, `611`, and `614`, whose stage players and fixture players are not yet represented as active tournament entrants.

## Backfill policy

Only backfill tournaments that are clearly generated by approved fixture tooling, for example via `tournaments.format_overrides.generated_by`:

- `scripts.amiga.tournament_builder`
- `site.public_html.amiga.ops.fixtures`

Do **not** backfill legacy Access/imported tournaments in this slice.

For each eligible generated tournament:

- Add any player found in `tournament_stage_players` as a `registered` entrant if missing.
- Add any non-null `tournament_fixtures.player_a_id` / `player_b_id` as a `registered` entrant if missing.
- Preserve existing entrant rows and statuses. Do not overwrite `withdrawn` or `replaced` rows unless you explicitly justify a safer behavior.
- Use deterministic `seed_no` where reasonable:
  - Prefer existing stage-player seed order when available.
  - Otherwise append after known seeds or leave `NULL` if safer.

The command should support:

- `--tournament-id N` to repair one tournament.
- no tournament id to repair all eligible generated tournaments.
- `--dry-run` to report what would be inserted without committing.

Suggested CLI name:

`python -m scripts.amiga fixtures backfill-entrants`

But choose another name if it better fits the module style.
